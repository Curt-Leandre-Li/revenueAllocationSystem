## Required Runtime Checks

1. `/allocation/constraints` saves a contract-ratio plan through the project
   scoped API: **PASS**. `ConstraintService.handleAction` maps `CONS-003` to
   `dvasApi.saveCurrentContractRatio`, and the backend writes
   `contract_ratio_plans` plus `contract_ratio_items`.

2. Saving `90% + 10%` is not hardcoded: **PASS**. The page converts entered
   percentages to decimal strings; `ContractRatioService.save` validates
   `ratio_sum = 1.000000` and computes amounts from the submitted
   `total_revenue`. Party ids and names come from backend party DTOs.

3. `/allocation/constraints` re-GETs backend state after save/clear:
   **PASS**. `mutateBackendAndRefresh` calls `refreshStoreFromBackend`, which
   reloads contract-ratio and allocation summary DTOs. The page uses returned
   `calculated_amount`, `ratio`, `amount_source`, and `can_simulate`.

4. Summary returns contract-ratio readiness fields: **PASS**.
   `ContractRatioService.summary` returns `contract_ratio_configured`,
   `contract_ratio_sum`, `data_provider_pool_ratio`,
   `data_provider_revenue_pool`, `non_data_contract_amount`,
   `weight_task_id`, `weight_sum`, and `blocking_reasons`.

5. Frontend displays backend fields rather than locally calculating allocation
   pool values: **PASS**. `buildConstraintsPage` and `buildSimulationPage`
   read contract-ratio/summary/result DTO fields. Local draft math is limited
   to input validation and does not replace backend returned amounts.

6. Simulation uses `data_provider_revenue_pool × MD-DShap normalized_weight`:
   **PASS**. `AllocationService.simulate_contract_ratio` computes non-data
   rows from `CONTRACT_RATIO` items and provider rows from
   `MD_DSHAP_WEIGHT`, then reconciles tail delta.

7. Final result includes all subjects: **PASS**. Backend returns non-data
   `CONTRACT_RATIO` rows plus data-provider `MD_DSHAP_WEIGHT` rows. Frontend
   result rows display `amount_source`, `contract_ratio`,
   `base_pool_amount`, and `final_amount`.

8. JSON/CSV/Markdown exports include contract-ratio plan fields:
   **PASS**. `ReportService._json_payload`, `_csv_files`, and
   `_markdown_content` include `contract_ratio_plan`,
   `contract_ratio_items`, `contract_ratio`, `base_pool_amount`,
   `amount_source`, and `non_data_contract_amount`.
