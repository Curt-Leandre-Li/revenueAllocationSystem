# P0/P1 边界与验收自检 Skill

## 适用场景

当前端页面、接口接入、图表、导出、状态机或交互整改完成后使用本 skill 做收口验收。目标是防止前端越界实现 P1、伪装后端能力、隐藏计算来源或弱化非法律结算边界。

## P0 必须覆盖

- 本地操作员模式，操作人显示为 `local_operator`。
- 演示数据选择和 JSON 上传。
- 数据接入、资源识别、参与方管理。
- 质量评估。
- 数元计量。
- 贡献度与效用计算。
- MD-DShap 权重计算。
- 收益分配模拟。
- 合同约束管理。
- Markdown / CSV / JSON / JSONL 导出。
- 审计日志和快照追溯。

## P0 不包含

- 登录。
- 生产级 RBAC。
- PDF 导出。
- CSV/XLSX 批量导入。
- 异步队列和异步任务进度。
- 多租户。
- 真实财务付款。
- 电子签章、税务、银行系统。
- 法律结算指令、合同签署、权属确认、主管审批替代。

P1 能力可以显示为规划态，但不得伪装成 P0 已上线。

## 计算来源自检

- 所有计算是否由后端接口或内部服务触发。
- 前端是否没有新增收益分配、质量评分、权重归一化、约束调整、数元计量、效用计算。
- 页面指标是否来自后端字段或 chart DTO。
- 图表是否没有浏览器端业务推导。
- MD-DShap 是否只解释为权重层，不解释为最终法律分配比例。
- 收益分配是否展示合同优先分配、数据源收益池、权重分配、合同约束调整顺序。

## 导出自检

- P0 只展示 Markdown、CSV、JSON、JSONL。
- PDF 按钮必须为 P1 规划或暂未启用。
- 每个导出都展示文件清单、字段范围、生成时间。
- 每个导出都展示 `report_id` 和 `checksum`。
- 历史报告不能静默覆盖。
- 导出弹窗和报告预览必须包含：系统结果仅为模拟参考，非法律结算 / 非法定结算结果。

## 操作与状态自检

- 每个计算页都有前置条件检查。
- 每个重要操作有 loading、success、error、disabled。
- 高风险操作有二次确认和影响范围。
- 已确认项目默认只读。
- 已导出项目默认只读，允许查看、下载、审计和复制新版本。
- 重新评估、重新计算、重新导出生成新版本，不覆盖历史。
- 缺少后端能力时显示禁用/规划/缺口，不写假成功。

## 审计自检

- 重要操作有 audit log 入口。
- 计算类结果能追溯输入快照、参数快照、输出快照。
- 算法页显示算法版本、任务 ID、结果 ID、trace。
- 导出页显示 report/export 记录、checksum、生成时间。
- 失败状态显示 failure_reason 或 `trace_id`。

```text

- 接口：
- 图表：

P0 覆盖：
- 已通过：
- 未覆盖：

P1 边界：
- 灰显/规划：
- 已阻止假实现：

计算来源：
- 后端接口：
- 前端仅格式化：

导出与审计：
- report_id/checksum：
- 免责声明：

遗留缺口：
- 后端 DTO：
- 缺失接口：
- 未来实现任务：
```
