# 页面模板与按钮行为 Skill

## 适用场景

当新增、重建或整改 DVAS 业务页时使用本 skill。目标是让所有页面共享同一信息架构、按钮语义、错误表达、trace 展示和验收骨架。

## 标准页面骨架

每个 P0 业务页按以下结构组织：

1. 页面标题区：页面名称、所属模块、项目状态、风险说明。
2. 前置条件检查区：缺什么、影响什么、下一步去哪。
3. 操作按钮区：主按钮、次按钮、危险按钮、导出按钮。
4. 主数据区：表格、卡片、图表或流程视图。
5. 详情区：抽屉或弹窗，承载明细、表单、对比、预览。
6. Trace 区：摘要表 + 展开详情，显示输入快照、参数快照、输出快照、算法版本、失败原因。
7. 空状态区：说明当前为什么为空，并给出下一步动作。
8. 异常区：字段级错误、业务规则错误、计算失败详情、后端缺口或 P1 不可用/未授权说明。

不要只写“暂无数据”。空状态必须指向下一步，例如“请先上传 JSON 输入文件”或“请先完成质量评估”。

## 按钮需求字段

核心按钮必须能追溯到以下字段：

- 功能编号。
- 按钮名称。
- 入口路径。
- 前置条件。
- 页面元素。
- 输入字段。
- 业务规则。
- 操作流程。
- 输出结果。
- 状态变化。
- 权限控制。
- 异常处理。
- 日志记录。
- 验收标准。

已有按钮以 `docs/ui/button_interaction_matrix.md` 为准；路由和字段以 `docs/ui_route_field_action_matrix.md` 为准。

## 按钮类型

| 类型 | 用法 | UI 行为 |
| --- | --- | --- |
| view | 查看详情、预览、trace、日志 | 可打开抽屉或详情页，只读 |
| create | 选择演示数据、新增参与方、新增合同比例项 | 表单或确认弹窗，成功后刷新后端数据 |
| update | 参数、权重、关系、金额配置 | 表单校验，已确认/已导出时禁用 |
| calculate | 质量、计量、效用、MD-DShap、分配 | 先展示前置检查，再调用后端计算 |
| export | Markdown、CSV、JSON、JSONL、审计日志 | 展示导出范围、免责声明、report/checksum 规则 |
| highRisk | 停用、锁定、不可逆确认 | 二次确认，展示影响范围和审计记录 |
| p1 | PDF、登录、RBAC、异步进度 | 标注 P1；后端不可用时显示不可用，不伪造成功 |

## 前置条件检查区

前置条件检查区至少展示：

- 当前阶段。
- 已满足项。
- 未满足项。
- 阻塞原因。
- 建议下一步路由。
- 相关对象 ID 或快照 ID，仅在技术详情或 trace 中展示。

前端不得重新计算前置条件。优先使用后端 `preconditions` 或页面 DTO。

## 表单与字段错误

- 字段级错误绑定具体字段，例如金额为负、权重合计异常、JSON 必填字段缺失。
- 业务规则错误放在表单顶部或前置条件区，例如“合同比例合计必须为 1.000000”或“未保存合同比例方案”。
- 计算失败必须显示失败阶段、失败原因、`trace_id` 或日志入口。
- 后端错误信封中的 `code`、`message`、`trace_id`、`field_errors` 需映射到页面。

## Trace 区

重要计算页必须有 trace 区：

- 质量评估：assessment、score detail、参数快照、证据。
- 数元计量：metering、detail、公式、参数版本。
- 贡献度与效用：contribution、utility、utility_trace。
- MD-DShap：task、result、marginal_trace、algorithm_audit_snapshot。
- 合同分配规则：contract_ratio_plan、contract_ratio_item、比例合计、可模拟状态。
- 收益分配模拟：allocation_result、合同比例方案、收益池、金额来源、尾差处理。
- 报告与审计：report_id、checksum、file list、snapshot refs。

默认只显示摘要，展开后展示技术细节。

## 只读与版本规则

- 已确认项目不能直接改输入、参与方、参数、约束或算法结果。
- 已导出项目不能静默覆盖报告、结果或快照。
- 重新评估、重新计算、重新导出必须生成新版本。
- 无后端复制新版本能力时，不做前端假复制。

## 页面输出检查

每个页面完成后，用以下问题自检：

- 页面标题区是否清楚标明模块、状态和风险边界。
- 前置条件是否可见且能引导下一步。
- 主按钮和危险按钮是否有明确禁用、loading、success、error 状态。
- 空状态是否给出下一步动作。
- 计算页是否有 trace 摘要和详情。
- 导出页是否展示 `report_id`、`checksum`、生成时间和免责声明。
- P1 能力是否没有被写成 P0 成功功能。
