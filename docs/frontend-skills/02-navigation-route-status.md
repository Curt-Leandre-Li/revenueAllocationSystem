# 导航、路由与状态机 Skill

## 适用场景

当前端重建左侧菜单、顶部栏、路由表、项目状态条、按钮启用逻辑或只读规则时使用本 skill。目标是一次性按新版导航结构收敛，而不是沿用旧路径逐页拼接。

## 固定左侧导航

左侧导航必须使用以下结构，菜单标签不显示阿拉伯数字前缀：

| 一级导航 | 二级页面 | 路由 | 阶段 |
| --- | --- | --- | --- |
| 系统首页 | 无二级页面 | `/dashboard` | P0 |
| 数据管理 | 数据接入管理 | `/data/ingestion` | P0 |
| 数据管理 | 数据资源管理 | `/data/resources` | P0 |
| 数据管理 | 参与方管理 | `/data/parties` | P0 |
| 数元贡献度计量 | 质量评估管理 | `/metering/quality` | P0 |
| 数元贡献度计量 | 数元计量管理 | `/metering/shuyuan` | P0 |
| 数元贡献度计量 | 贡献度与效用计算 | `/metering/utility` | P0 |
| 收益分配计算 | MD-DShap 计算管理 | `/allocation/md-dshap` | P0 |
| 收益分配计算 | 合同分配规则 | `/allocation/constraints` | P0 |
| 收益分配计算 | 收益分配模拟 | `/allocation/simulation` | P0 |
| 报告生成与导出 | 无二级页面 | `/reports` | P0/P1 |
| 系统管理 | 参数配置 | `/system/parameters` | P0 |
| 系统管理 | 用户与权限管理（P1） | `/system/users` | P1 |
| 系统管理 | 审计日志管理 | `/system/audit` | P0 |

系统首页是一级导航页面，不设置二级窗口。项目总览、流程入口、风险提示、一键计算是首页内部区块，不进入 `nav_menu`，不作为独立路由，不单独配置权限菜单。

## 顶部栏固定信息

顶部栏必须展示：

- 系统名称：数据收益分配系统 V1.2。
- 当前项目。
- 当前状态。
- 当前操作人，P0 默认为 `local_operator`。
- 风险提示入口或风险提示可见状态。

风险提示不得被主题、空状态、加载失败或窄屏布局隐藏。

## 项目状态条

项目状态条固定展示以下阶段：

```text
草稿 -> 已接入 -> 已评估 -> 已计量 -> 已计算效用 -> 已计算权重 -> 已分配 -> 已确认 -> 已导出
```

状态条用于表达流程位置，不用于在前端计算阶段结果。阶段结果必须来自后端项目状态、任务状态或聚合接口。

## 状态门禁

| 目标动作 | 最低前置状态 | 失败处理 |
| --- | --- | --- |
| 质量评估 | 已接入 | 提示先选择演示数据或上传 JSON |
| 数元计量 | 已评估 | 提示先完成质量评估 |
| 贡献度与效用计算 | 已计量 | 提示先完成数元计量 |
| MD-DShap 计算 | 已计算效用 | 提示先完成贡献度与效用计算 |
| 收益分配模拟 | 已计算权重 | 提示先完成 MD-DShap 权重计算 |
| 锁定分配方案 | 已分配 | 提示先执行收益分配模拟 |
| 报告导出 | 已分配或已确认 | 提示先生成分配结果 |
| 审计日志导出 | 存在日志 | 提示暂无可导出的审计日志 |

## 按钮启用逻辑

- 优先使用后端返回的 `preconditions`、`available_actions`、`disabled_actions`。
- 前端只做展示层禁用和提示，不重新判定业务合法性。
- 高风险操作必须二次确认：删除/停用数据包、停用参与方、清空合同比例方案、锁定分配方案、覆盖风险较高的参数变更。
- 计算按钮先展示前置条件检查，再调用后端计算接口。
- 导出按钮先展示导出范围、文件类型、免责声明、`report_id` / `checksum` 规则，再调用后端导出接口。

## 已确认和已导出只读规则

- 已确认项目：参数、输入、约束、参与方、资源关系默认只读。
- 已导出项目：不得静默覆盖报告或历史结果。
- 如需修改，提供“复制新版本”入口；没有后端复制接口时只展示规划态，不做前端假复制。
- 查看、下载、审计追溯保持可用。

## P1 页面策略

- `/system/users` 在 P1 闭环中展示真实用户、角色和权限数据。
- PDF、登录、RBAC、模板导入、任务进度和历史报告必须调用后端 P1 接口。
- P1 页面不得写入前端假数据，不得模拟生产级权限成功。

## 自检清单

- 左侧菜单是否完全匹配新版结构。
- 系统首页是否只有 `/dashboard` 一个路由。
- 顶部栏是否展示系统名称、项目、状态、操作人和风险入口。
- 状态条是否按固定九阶段展示。
- 按钮禁用是否来自后端门禁或明确的后端缺口。
- 已确认、已导出项目是否只读且不能覆盖历史。
