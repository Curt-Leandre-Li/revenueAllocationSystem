# 大厂后台设计参考 Skill

## 适用场景

当 DVAS 前端需要确定整体风格、布局密度、组件模式、后台交互方式或视觉语言时使用本 skill。目标是借鉴成熟企业后台的通用设计逻辑，而不是复制某一家产品。

## 设计定位

DVAS 应像“企业级数据资产 / 收益分配控制台”：

- 信息密度高。
- 状态标签明确。
- 表格和抽屉成熟。
- 流程状态可追踪。
- 风险边界清晰。
- 审计、trace、snapshot、checksum 易查。

不做花哨大屏，不做营销式 hero，不做重动画，不做过度装饰。

## 可借鉴方向

- 云控制台：左侧导航、顶部项目上下文、资源列表、操作列、状态标签。
- Ant Design Pro 类后台：表格、筛选、表单、弹窗、抽屉、详情页的成熟模式。
- 数据中台 / BI 看板：首页流程进度、指标总览、质量概况、结果图表。
- 审计系统：trace、快照、checksum、操作日志、失败原因和回溯入口。

借鉴交互逻辑，不直接照搬品牌、配色、图标、页面结构或代码。

## 布局规则

- 左侧导航固定，顶部显示项目上下文。
- 内容区以页面标题、前置条件、操作条、主数据、详情/trace 组织。
- 表格是主工作区，不要把所有内容拆成孤立卡片。
- 卡片用于指标、状态、风险提示和重复实体摘要。
- 抽屉用于详情、trace、快照、预览、字段说明。
- 弹窗用于确认、短表单、高风险二次确认。
- 导出确认必须独立弹窗或抽屉，展示文件范围和免责声明。

## 视觉规则

- 主色保持克制，优先使用企业后台常见蓝/青/中性色系。
- 状态色语义固定：成功、警告、错误、禁用、处理中。
- 不使用复杂渐变、炫光背景、沉浸大屏或重装饰插画。
- 信息层级靠间距、字号、分组和标签，不靠夸张视觉效果。
- 中文管理后台文案要短，按钮动词明确。

## 组件模式

| 场景 | 推荐组件 |
| --- | --- |
| 主数据浏览 | 表格 + 筛选 + 操作列 |
| 页面摘要 | 指标卡 + 状态标签 |
| 业务流程 | 步骤条 + 前置条件卡 |
| 明细查看 | 右侧抽屉 |
| 高风险动作 | 二次确认弹窗 |
| 计算过程 | trace 摘要表 + 展开详情 |
| 审计回溯 | 时间线 + 日志表 + 快照抽屉 |
| P1 能力 | 灰显入口 + 规划说明 |

## 模块风格重点

- 系统首页：项目总览、流程入口、风险提示、一键计算管线，保持工作台感。
- 数据管理：资源、数据包、参与方用高密度表格和详情抽屉。
- 数元贡献度计量：强调前置条件、计算结果、证据和 trace。
- 收益分配计算：强调收益池、权重、合同约束、模拟参考边界。
- 报告生成与导出：强调文件清单、字段范围、版本、checksum。
- 系统管理：强调参数版本、P1 权限边界、审计检索。

## 自检清单

- 页面是否像可长期使用的企业后台，而不是展示大屏。
- 主要业务对象是否能通过表格扫描、筛选和打开详情。
- 计算和导出是否有清楚的状态、错误和追溯入口。
- 风险说明是否可见但不过度干扰主流程。
- P1 能力是否通过灰显/规划态表达，而不是假实现。
