## 一、软件概述

### 1.1 软件名称

数据收益分配系统 V1.0。

### 1.2 软件用途

数据收益分配系统用于完成数据接入、数据资源查看、参与方维护、质量评估、数元计量、贡献度与效用计算、MD-DShap 权重计算、合同比例配置、收益分配模拟、报告导出和审计追溯等操作。用户可以在页面上查看每一步的处理状态，并将计算结果导出为 Markdown、CSV、JSON 或 JSONL，用于本地演示、业务复核和审计说明。

本系统用于收益分配模拟。系统根据质量、使用、贡献、效用和权重信息生成分配参考结果，说明数据资源和参与方在模拟收益分配中的作用。系统不生成付款指令，也不替代合同、法律、财务或主管单位审批流程。

### 1.3 适用对象

- 数据资源管理人员。
- 数据运营人员。
- 数据收益分配业务人员。
- 算法审核人员。
- 审计查看人员。
- 软著演示或本地测试人员。

### 1.4 使用边界

本系统输出结果仅作为收益分配模拟参考，不构成法律结算、财务付款、合同履约、资产评估、正式审计结论或主管单位审批依据。用户在报告、导出文件和审计说明中看到的金额、权重、比例、校验和、快照等内容，均用于模拟展示和追溯说明。

## 二、运行环境与基本配置

### 2.1 硬件环境

| 配置项 | 推荐配置 |
| --- | --- |
| CPU | Intel i3 或同等性能以上处理器 |
| 内存 | 8GB 及以上 |
| 硬盘 | 10GB 及以上可用空间 |
| 网络 | 本地演示可不依赖外网，前后端服务需能在本机端口访问 |

### 2.2 软件环境

| 环境项 | 说明 |
| --- | --- |
| 操作系统 | macOS、Linux 或 Windows。当前截图和手册基于本地 macOS 开发环境生成。 |
| 后端运行环境 | Python。本项目后端以 `backend.dvas.server` 启动，本地 HTTP 服务由 `ThreadingHTTPServer` 和 `DvasApplication` 提供。 |
| 前端运行环境 | Node.js、React、Vite。前端源码位于 `ui_prototype/`。 |
| 数据存储 | P0 本地演示默认使用 JSON runtime state；PostgreSQL 位于 `db/`，作为参考数据库或验收数据库。 |
| 浏览器 | Chrome、Edge 等现代浏览器。 |

### 2.3 启动后端服务

用户在本地终端进入项目根目录后，执行以下命令启动后端服务：

```bash
cd <项目根目录>
python3.12 -m backend.dvas.server
```

后端默认监听 `http://127.0.0.1:8000/api/v1`。如果本机 Python 命令不同，可根据本地环境使用等效的 Python 3.12 命令。

### 2.4 启动前端服务

用户在另一个终端进入前端目录后，执行以下命令：

```bash
cd <项目根目录>/ui_prototype
npm install
npm run dev
```

当前 `package.json` 中的开发命令为 `vite --host 127.0.0.1`。前端默认通过 `VITE_API_BASE_URL` 或 `VITE_DVAS_API_BASE_URL` 读取后端地址；未配置时使用 `http://127.0.0.1:8000/api/v1`。

### 2.5 访问系统

前端服务启动后，用户可以在浏览器访问：

```text
http://127.0.0.1:5173/
```

如果本机 Vite 使用其他端口，应以终端输出的访问地址为准。

### 2.6 登录或进入系统

当前页面包含登录入口。用户输入由系统管理员分配的账号后，可以进入系统首页。登录和用户权限能力属于当前本地演示与 P1 扩展能力，生产环境应替换默认开发账号和密码策略。P0 业务以本地演示、数据接入、计算、报告和审计记录为主。

## 三、软件整体操作流程

用户使用本系统时，一般按以下流程操作：

1. 打开浏览器进入系统地址。
2. 登录或进入系统首页。
3. 选择演示数据或上传 JSON 数据包。
4. 查看数据资源。
5. 查看和维护参与方。
6. 执行质量评估。
7. 执行数元计量。
8. 执行贡献度与效用计算。
9. 执行 MD-DShap 权重计算。
10. 配置合同比例方案。
11. 执行收益分配模拟。
12. 导出报告。
13. 查看审计日志。

## 五、数据管理

### 5.1 数据接入管理

#### 5.1.1 功能说明

数据接入管理页面用于选择演示数据、上传 JSON 数据包、查看数据包列表、查看校验状态、预览数据包内容以及查看上传失败原因。用户完成数据接入后，系统会生成输入快照，并为后续数据资源识别、参与方维护和质量评估提供基础数据。

#### 5.1.2 操作入口

用户点击左侧导航“数据管理”，再进入“数据接入管理”。页面路由为 `/data/ingestion`。

#### 5.1.3 操作步骤

1. 用户进入数据接入管理页面。
2. 用户点击“使用示例数据”或“上传 JSON”。
3. 系统进行格式和字段校验。
4. 校验通过后，系统生成数据包和输入快照。
5. 校验失败时，用户点击“失败详情”查看错误字段和原因。
6. 用户可以点击“预览”查看数据包摘要。

## 七、收益分配计算

### 7.1 MD-DShap 计算管理

#### 7.1.1 功能说明

MD-DShap 计算管理页面用于计算数据源主体之间的贡献权重。MD-DShap 输出的是权重参考，不是最终付款指令。非数据主体不进入 MD-DShap 权重池，系统会在页面中展示参与方集合、任务状态、权重结果、边际贡献和算法审计说明。

#### 7.1.2 操作步骤

1. 用户进入“收益分配计算 - MD-DShap 计算管理”。
2. 用户查看参与方集合，确认数据源主体进入权重池。
3. 用户确认非数据主体不进入权重池。
4. 用户点击“启动计算”。
5. 系统输出参与方权重。
6. 用户查看权重、边际贡献或算法审计说明。

#### 7.1.3 按钮说明

| 按钮名称 | 功能说明 | 点击后效果 | 当前可用性 |
| --- | --- | --- | --- |
| 查看审计 | 查看算法审计说明。 | 打开审计说明或跳转报告导出。 | P0 可用 |
| 查看边际贡献 | 查看边际贡献 trace。 | 打开边际贡献明细。 | P0 可用 |

### 7.2 合同分配规则

#### 7.2.1 功能说明

当前系统保留独立的“合同分配规则”页面。该页面的主流程用于维护项目合同比例方案，包括总收益、数据源收益池比例和非数据主体合同比例。保存合同比例方案后，收益分配模拟会先按合同比例划分非数据主体合同金额和数据源主体收益池，再使用 MD-DShap 归一化权重分配数据源主体收益池。

## 十一、常见问题与注意事项

### 11.1 为什么系统结果不是法律结算？

本系统定位为数据收益分配模拟与审计说明系统。系统输出用于演示和参考，不构成法律结算、财务付款、合同履约或主管单位审批依据。

### 11.2 上传文件失败怎么办？

用户可以点击“失败详情”查看字段错误、格式错误或校验失败原因。修正 JSON 文件后，重新上传即可。上传失败时系统不会生成有效数据包。

### 11.3 为什么某些按钮是灰色？

按钮灰色通常表示前置条件未满足、当前用户无权限、后端接口未接入或该能力属于 P1 预留。例如资源摘要导出、纯算法结果导出、收益分配确认书等能力在当前 P0 中不应伪造成已实现。

### 11.4 为什么未完成前置步骤不能执行计算？

质量评估、数元计量、效用计算、MD-DShap 和收益分配模拟之间有顺序依赖。系统需要先获得数据包、资源主体关系、质量结果、计量结果、效用结果和权重结果，才能生成可追溯的收益分配模拟结果。

### 11.5 为什么非数据主体不进入 MD-DShap？

MD-DShap 用于计算数据源主体之间的贡献权重。非数据主体如运营服务方、合同服务方等不进入权重池，当前主流程通过合同比例方案参与收益分配模拟。

### 11.6 为什么报告导出需要先完成收益分配？

报告需要引用项目、数据包、资源、参与方、质量评估、数元计量、效用、MD-DShap 和分配结果等信息。如果收益分配未完成，报告无法形成完整的模拟参考内容。

### 11.7 为什么已锁定或已导出的结果不能直接修改？

已锁定或已导出的结果需要保留历史版本和审计追溯。系统应通过新版本或重新计算方式生成新的结果，而不是静默覆盖历史记录。

### 11.8 当前 P0 与 P1 功能有什么区别？

P0 重点支持本地演示、JSON 上传、数据资源管理、质量评估、数元计量、贡献效用、MD-DShap、合同比例、收益分配模拟、Markdown/CSV/JSON/JSONL 导出和审计追溯。P1 可扩展登录与 RBAC、PDF、CSV/XLSX 模板导入、异步任务、历史报告管理和更完整的权限控制。
