# 管理员手册

## 1. Manual Purpose

This manual describes planned operation and archive responsibilities for the data revenue allocation system. It is a Phase 0 baseline manual and must be completed with actual commands after implementation exists.

## 2. Administrator Scope

In the current MVP, administrator means the local demo operator and archive maintainer.

The MVP does not include a multi-user account system, role-based permission system, or production authentication module.

## 3. Planned Responsibilities

- Prepare simulated input packages.
- Run validation and pipeline commands after implementation exists.
- Confirm that generated reports are labeled as simulated.
- Archive exported JSON and report artifacts.
- Record actual test commands and evidence after coding.
- Ensure no real hospital data is inserted into sample artifacts.

## 4. Directory And Artifact Expectations

| Artifact | Purpose | Current Status |
| --- | --- | --- |
| `docs/` | Product and architecture freeze documents. | Available after Phase 0. |
| `docs/deliverables/` | Formal R&D archive documents. | Available after Phase 0. |
| Input samples | Simulated package data. | To be created during coding. |
| Report output | Audit-readable generated report. | To be created during coding. |
| Test evidence | Commands, logs, and screenshots if available. | To be created during coding. |

## 5. Planned Operation Steps

1. Confirm the repository is on the intended branch.
2. Confirm no unapproved product code or dependency change is present.
3. Prepare simulated input data.
4. Run validation.
5. Run the pipeline.
6. Review allocation reference and warnings.
7. Generate report artifact.
8. Archive report and test evidence.

Actual command lines are intentionally not listed yet because product runtime files do not exist in Phase 0.

## 6. Data Safety Rules

- Use simulated data only.
- Do not describe sample data as real hospital data.
- Do not describe allocation reference as legal settlement result.
- Do not remove confirmation-required warnings from reports.
- Do not restore MAR, Effective DU, or `token_weighting` as main-process terms.

## 7. Maintenance Rule

After coding, this manual must be updated with verified local run commands, configuration notes, generated artifact paths, and troubleshooting steps.
