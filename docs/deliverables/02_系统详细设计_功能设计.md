# 系统详细设计：功能设计

## 设计定位

本文为 V1.2 功能设计归档摘要。详细设计以根目录
`数据收益分配系统_系统详细功能设计_V1.1_导航结构更新版.md` 为准。

## 功能分层

- 入口与状态层：系统首页、流程入口、风险提示、一键计算。
- 数据准备层：数据接入管理、数据资源管理、参与方管理。
- 质量与计量层：质量评估管理、数元计量管理、贡献度与效用计算。
- 算法权重层：MD-DShap 计算管理，基础 Shapley 仅作基线校验。
- 分配约束层：收益分配模拟、合同约束管理。
- 报告审计层：报告生成与导出、审计日志管理。
- 系统支撑层：参数配置、用户与权限管理（P1）。

## 关键设计规则

- 计算按钮必须展示前置条件检查。
- 重新评估、重新计量、重新计算、重新导出均生成新版本。
- MD-DShap 输出权重，不直接形成付款指令。
- 非数据贡献主体默认通过合同优先或约束处理。
- 导出字段固定化，P0 生成 Markdown/CSV/JSON/JSONL。
- 审计快照贯穿输入、参数、输出、报告和算法过程。

系统结果仅为模拟参考，非法律结算 / 非法定结算结果。
