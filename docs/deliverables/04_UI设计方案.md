# UI设计方案

## 1. UI Goal

The UI presents a lightweight demonstration workflow for the data revenue allocation system. It helps reviewers inspect simulated inputs, validation results, pipeline artifacts, contribution references, contract constraints, and reports.

## 2. Page List

| Page | Purpose | Main Components |
| --- | --- | --- |
| Input Package | Review or edit simulated input package. | `InputPackageEditor`, participant table, Data Unit table, constraint panel. |
| Validation Results | Show validation errors and warnings. | `ValidationSummary`, error list, warning list. |
| Pipeline Review | Show pipeline step status and artifact summaries. | `PipelineStepList`, artifact summary cards. |
| Signal Details | Inspect quality, metering, utility, DAUS, and Shapley references. | `SignalTable`, contribution reference table. |
| Contribution And Allocation | Review participant allocation reference and constraints. | `AllocationReferencePanel`, `ConstraintWarningPanel`. |
| Audit Report | Read generated report artifact. | `AuditReportView`, trace appendix. |
| Export | Export JSON and report artifacts. | `ExportActions`. |

## 3. Navigation Flow

```text
Input Package
-> Validation Results
-> Pipeline Review
-> Signal Details
-> Contribution And Allocation
-> Audit Report
-> Export
```

Validation errors keep the user on Validation Results until the input is corrected.

## 4. UI States

| State | Meaning | UI Behavior |
| --- | --- | --- |
| Empty | No package loaded. | Show input entry area. |
| Draft | Package is being edited. | Allow validation. |
| Invalid | Validation errors exist. | Disable pipeline run. |
| Ready | Validation has no errors. | Allow pipeline run. |
| Running | Pipeline is calculating. | Show running state and lock repeated run. |
| Complete | Artifacts exist. | Enable report and export. |
| Warning | Output needs confirmation. | Show warning labels. |
| Failed | Unexpected error occurred. | Show structured error. |

## 5. Information Design

- Simulated-data label must be visible on input, output, and report pages.
- Shapley must be shown as contribution reference, not final allocation.
- Final allocation reference must show contract constraint effects.
- Medical-looking examples must be labeled as generic simulated examples.
- UI must not calculate independent allocation math.

## 6. Component Split

- `WorkflowShell`: page layout and navigation.
- `InputPackageEditor`: package editing and sample load.
- `ValidationSummary`: error and warning display.
- `PipelineStepList`: step status display.
- `SignalTable`: quality, metering, utility, and contribution tables.
- `ContributionReferenceTable`: DAUS and Shapley contribution reference display.
- `AllocationReferencePanel`: final allocation reference display.
- `ConstraintWarningPanel`: cap, minimum guarantee, and confirmation warnings.
- `AuditReportView`: report sections and audit trace.
- `ExportActions`: JSON/report export entry points.

## 7. Phase 0 Boundary

This file defines UI design before implementation. It does not claim that the frontend has already been built.
