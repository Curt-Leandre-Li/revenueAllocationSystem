# UI 设计方案

## 设计目标

- 1440x900 桌面端 Web 管理后台。
- 中文界面。
- B 端 SaaS / 咨询公司风格。
- 白底深蓝、浅蓝信息区、灰色分隔线。
- 每页显著展示：系统结果仅为模拟参考，非法定/非法律结算结果。

## 左侧导航

- 系统首页：单一一级入口；项目总览、流程入口、风险提示、一键计算为系统首页内部区块，不作为二级菜单、二级路由或独立权限节点
- 数据管理：数据接入管理、数据资源管理、参与方管理
- 数元贡献度计量：质量评估管理、数元计量管理、贡献度与效用计算
- 收益分配计算：MD-DShap 计算管理、收益分配模拟、合同约束管理
- 报告生成与导出
- 系统管理：参数配置、用户与权限管理（P1）、审计日志管理

## 交互要求

- 计算按钮先展示前置条件检查。
- 导出弹窗展示导出文件、字段范围、报告版本和免责声明。
- 重新计算提示不会覆盖历史结果。
- 已确认/已导出项目编辑按钮置灰，并提供复制新版本入口。
- 空状态说明下一步操作。
- 异常状态定位字段、前置条件或业务规则。

肺癌早筛数据收益分配示例项目可作为示例项目，但不得写成系统唯一场景。
