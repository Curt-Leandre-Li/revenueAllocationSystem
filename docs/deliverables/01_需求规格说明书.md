# 需求规格说明书

## 系统定位

系统名称：数据收益分配系统 V1.2。英文简称：DVAS，Data Value Allocation
System。系统定位为数据收益分配模拟与审计说明系统。

系统输出仅为收益分配模拟参考，不构成法律结算、财务付款、合同履约或主管单位
审批结果。

## 范围

P0 包含本地操作员、演示数据/JSON 上传、数据接入、资源识别、参与方管理、
质量评估、数元计量、贡献度与效用计算、MD-DShap 权重计算、收益分配模拟、
合同分配规则、Markdown/CSV/JSON/JSONL 导出、审计日志和快照追溯。

P1 扩展登录与 RBAC、PDF 导出、CSV/XLSX 模板导入、异步任务进度、历史报告
管理和更完整权限控制。

## 最新导航

- 系统首页：单一一级入口；项目总览、流程入口、风险提示、一键计算为系统首页内部区块，不作为二级菜单、二级路由或独立权限节点
- 数据管理：数据接入管理、数据资源管理、参与方管理
- 数元贡献度计量：质量评估管理、数元计量管理、贡献度与效用计算
- 收益分配计算：MD-DShap 计算管理、合同分配规则、收益分配模拟
- 报告生成与导出
- 系统管理：参数配置、用户与权限管理（P1）、审计日志管理

## 核心需求

- MD-DShap 是默认贡献权重计算策略。
- 基础 Shapley 仅作为小规模 `baseline_check`。
- 非数据贡献主体默认不进入 MD-DShap 算法池。
- 单数据源主体权重为 1，并披露单主体简化分配。
- 权重合计归一化为 1，展示 6 位小数。
- 重新计算不得覆盖历史 task/result/trace。
- 每次导出必须生成 `report_id` 和 `checksum`。

详细需求以根目录 V1.4 需求规格说明书为准。
